开源社区里面常用的 Git 操作

发表于 2017-05-16 |

Git Commands in Github & Gitlab & Git [ 这篇文章不适合 Git 新手 ]

最常用的 Git 操作命令 一般有 git add, git commit, git fetch, git rebase, git reset, git push, git branch, git log, git rm, git checkout, git remote 以及加在这些命令后面的参数，然后其它的我好像也没有怎么用过了。我不会讲解每个命令的具体用法，只说我是如何在工作中使用它们的，并且如何用这些命令来完成看似比较复杂的开源社区以及在公司里面如何同时在 10 个左右的分支上同时开发项目的，以及在子分支上建立 subtask 再一步一步 merge 到 master / dev / trunk 分支上。

git add

太简单，不想说了。

git commit

太简单，不想说了。

git fetch

每天早上打开电脑的第一件事，一般就是执行该命令。

我经常这样做：

git fetch upstream master 【社区】
git fetch 【公司】

git rebase

每天早上打开电脑的第二件事，一般就是执行这条命令；但是你可以不立马这样做，什么时候做呢？在你提交到远程分支之前进行 rebase, 如果开发的 patch 比较大，那么需要定期 rebase 远程分支代码到本地开发分支，不然最后一次性 rebase 容易导致爆炸性冲突，麻烦呢。 所以养成定期 rebase 的习惯不失为一件坏事情。

我经常这样做：

git rebase origin dev 【公司】
git rebase upstream master 【社区】
### git reset ### 对于这个命令，我一般用它来 squash commit logs。 当然你也可以用 git rebase -i hashcode 去 squash logs, 不过用 rebase 的方式太过于麻烦。 所以用 ```git reset --soft HEAD~n``` [n belongs to a number], 当执行完这个命令后，不要忘记 ```git commit```, 不然你所有的改动将会丢失哦 ~ 小心点。 我经常这样做： 
git reset --soft HEAD~3
git reset --soft HEAD~8
### git push ### 我一般用它完成本地代码推送到远程服务器上操作，还有就是新建一个分支，推送分支到远程服务器。 ### git log ### 查看提交历史，我主要用来检查 rebase 是否成功，hashcode 的值是否和远程的一致，如果一致则说明 rebase 成功，如果不一致，说明 rebase 失败。 ### git rm ### ### git checkout ### ### git remote ### # 社区开发的 7 个步骤 [以 Flink 为例 ]： **第一步**： 
git remote add upstream https://github.com/apache/flink

第一步如果加错了，可以执行如下的命令删除即可

git remote remove upstream

第二步：

git remote -v

第三步：

git fetch upstream master

第四步：

git rebase upstream/master

第五步：
git log 查看是否更新成功

第六步：

git push origin master

第七步：
打开自己的 github 查看是否成功
