这篇论文的作者见本文最后的图片，Leslie Lamport， 2013 年图灵奖得主。 这篇论文成为计算机科学史上被引用最多的文献， 他为 并发系统的规范与验证 研究贡献了核心原理。

在正式进入这篇论文之前，首先需要说明几个概念。

Hasse diagram

该图引用自维基百科


该图引用自维基百科


如图，翻译成中文叫做 哈斯图 ，用来表示一个 有序的 偏序集合。 从图中可以看出，对于 {x, y} 集合的三个子集 {Φ}, {x}, {y} 在哈斯图中是以一定的顺序去排序的。 并把该图记作：(S, ≤) 。其中 S 为这个图中的集合点，≤ 表示顺序关系。 如果有 (x < y)，那么就把这种情况叫做 y 覆盖 x。 绘制从 x 向上的线段或曲线。

[ 插入一句题外话，其实我们在日常计算机程序的开发中，可以把这里的 S 看成是，比如一个对文件系统操作的类，而这里的 ≤ 就可以看成是该类具有的操作，比如读取文件、上传文件、求文件大小、求 Block 数，等等。]

所以针对上面的两张图，其中对于第二张图，显然，(Φ < x)，(Φ < y)，(x < (x,y))等等。 同样，这也是一个空间上的图，有 ((x,z) < (x,y,z))。这样的一个图，也叫做偏序，英文叫做 ： partial order，其中 partial 翻译成中文的意思是：局部的意思，那么这个局部到底该怎么理解? 这里的意思是，在这个 S 的集合中，并不是每对元素都具有可比性，这是很显然的。 所以综上所述，哈斯图 是一个用来表示 有序的 偏序集合。 先记住这个概念，论文后面用到了。
